# 脳トレアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
Brain Training App（仮称）

### 1.2 目的
日々の脳トレを通じて、ユーザーの認知機能（記憶力、計算力、語彙力、論理思考力、反応速度）の維持・向上をサポートするWebアプリケーションの開発

### 1.3 対象ユーザー
- 脳トレを習慣にしたい一般ユーザー
- 認知機能の低下を心配する中高年層
- 学生・社会人の集中力・学力向上を目指す層

---

## 2. 技術スタック（推奨）

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | React + TypeScript | 型安全性、ECOSYSTEMの豊富さ |
| ビルドツール | Vite | 高速な開発体験 |
| ステート管理 | Zustand | 軽量でシンプル |
| UIコンポーネント | Tailwind CSS | 高速なスタイリング |
| バックエンド | Node.js + Express | フロントとの共有言語 |
| データベース | PostgreSQL + Prisma | リレーショナルデータに最適 |
| 認証 | NextAuth.js / JWT | 柔軟な認証機構 |
| デプロイ | Vercel (フロント) / Railway (バックエンド) | 簡単なデプロイ |

## 3. 機能要件

### 3.1 ユーザー管理機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| 新規登録 | メールアドレス / SNS認証（Google） | 高 |
| ログイン | メールアドレス + パスワード | 高 |

### 3.2 脳トレゲーム一覧

#### 3.2.1 記憶力トレーニング
| ゲーム名 | 概要 | レベル |
|----------|------|--------|
| カードマッチ | 2枚のカードの組み合わせを記憶してめくる | Easy / Normal / Hard |
| 数字記憶 | 表示された数字列を覚えて再入力 | 3桁〜8桁 |
| 色記憶 | 色の順序を記憶して再現 | 4色〜10色 |

#### 3.2.2 計算力トレーニング
| ゲーム名 | 概要 | レベル |
|----------|------|--------|
| 速算チャレンジ | 制限時間内に計算問題を解く | +−×÷ |
| 電卓パズル | 数字を組み合わせて目標値を作る | 3〜5個 |
| 数列完成 | パターンを理解し次にくる数字を当てる | Easy / Normal / Hard |

#### 3.2.3 語彙力トレーニング
| ゲーム名 | 概要 | レベル |
|----------|------|--------|
| 類義語チェック | 似た意味の言葉を選ぶ | Easy / Normal / Hard |
| 穴埋めクイズ | 文脈に合う言葉を選択 | Easy / Normal / Hard |
| 文字数パズル | 指定文字数で言葉を作成 | 3文字〜6文字 |

#### 3.2.4 論理思考トレーニング
| ゲーム名 | 概要 | レベル |
|----------|------|--------|
| パターン認識 | 図形の並びから次を当てる | Easy / Normal / Hard |
| 推理クイズ | 条件から正解を導く論理パズル | Easy / Normal / Hard |
| グリッドロジック | 条件からグリッドを埋める | 4×4〜8×8 |

#### 3.2.5 反応速度トレーニング
| ゲーム名 | 概要 | レベル |
|----------|------|--------|
| ターゲットクリック | 出現したターゲットを素早くクリック | Easy / Normal / Hard |
| 色文字マッチ | 文字の色と意味の不一致を判定（ストゥープ効果） | Easy / Normal / Hard |
| タイピングチャレンジ | 表示された文字列を高速入力 | Easy / Normal / Hard |

## 5. API設計（概要）

### 5.1 認証関連
```
POST   /api/auth/register    - 新規登録
POST   /api/auth/login       - ログイン
POST   /api/auth/logout      - ログアウト
POST   /api/auth/reset       - パスワードリセット
```

### 5.2 ゲーム関連
```
GET    /api/games             - ゲーム一覧取得
GET    /api/games/:id         - ゲーム詳細取得
POST   /api/scores            - スコア投稿
GET    /api/scores/me         - 自分のスコア履歴
GET    /api/scores/ranking    - ランキング取得
```

### 5.3 ユーザー関連
```
GET    /api/users/me          - 自分のプロフィール
PUT    /api/users/me          - プロフィール更新
GET    /api/users/me/stats    - 統計情報取得
```

## 6. データベース設計（概要）

### 6.1 主要テーブル

```sql
-- ユーザーテーブル
users
├── id (UUID, PK)
├── email (VARCHAR, UNIQUE)
├── password_hash (VARCHAR)
├── display_name (VARCHAR)
├── avatar_url (VARCHAR)
├── created_at (TIMESTAMP)
└── updated_at (TIMESTAMP)

-- ゲームテーブル
games
├── id (UUID, PK)
├── name (VARCHAR)
├── category (ENUM: memory, calculation, vocabulary, logic, reaction)
├── description (TEXT)
└── config (JSONB)

-- スコアテーブル
scores
├── id (UUID, PK)
├── user_id (UUID, FK)
├── game_id (UUID, FK)
├── score (INTEGER)
├── difficulty (ENUM: easy, normal, hard)
├── duration (INTEGER)
└── created_at (TIMESTAMP)

-- バッジテーブル
badges
├── id (UUID, PK)
├── name (VARCHAR)
├── description (TEXT)
└── criteria (JSONB)

-- ユーザーバッジテーブル
user_badges
├── user_id (UUID, FK)
├── badge_id (UUID, FK)
└── earned_at (TIMESTAMP)
```
